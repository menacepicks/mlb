# MLB Live Market Layer

This package adds the missing live-market layer for an MLB model:

- unifies DraftKings + FanDuel into one shared live schema
- keeps `full_game` and `innings` markets separate
- keeps milestone ladders separate from O/U
- keeps `player`, `team`, and `game` scopes separate
- prices count markets with Monte Carlo from model projections
- builds a daily market hub and split player/team/game boards

## What it writes

- `mlb_live_market_hub.csv`
- `mlb_live_scored_board.csv`
- `mlb_live_player_board.csv`
- `mlb_live_team_board.csv`
- `mlb_live_game_board.csv`

## Supported market families

- moneyline
- spread / run line
- game total runs
- team total runs
- team hits
- player hits
- total bases
- runs
- RBIs
- hits+runs+RBIs
- home runs
- pitcher strikeouts
- pitching outs
- earned runs
- hits allowed
- walks allowed
- stolen bases
- singles / doubles / triples
- first-inning run markets

## Inputs

### DraftKings

- raw JSON payloads shaped like `markets + selections (+ optional events)`
- or the already-normalized shared odds CSV/parquet generated by your existing DK adapter

### FanDuel

- raw event pages JSON + raw price rows JSON
- or the already-normalized shared odds CSV/parquet generated by your existing FD capture

### Projections

Optional CSV/parquet with columns such as:

- `scope`
- `segment`
- `market_family`
- `event_id`
- `participant`
- `team_name`
- `projection_mean`
- `projection_prob`

`projection_mean` is used for count-market Monte Carlo pricing.
`projection_prob` is used directly for probability markets.

## Important notes

- This package is designed to sit **on top of** your existing capture layer.
- It does not assume a single hard-coded sportsbook endpoint.
- It can score a market from model projections when those projections are supplied.
- It does **not** invent missing historical model features. It expects your model layer to provide projections or probabilities where available.
